# 步骤1：JVM基础 - 运行时数据区

## 1.1 JVM概述

### 1.1.1 JVM是什么

```
JVM（Java Virtual Machine）：Java虚拟机

职责：
- 加载字节码（.class文件）
- 执行字节码
- 管理内存
- 垃圾回收

跨平台原理：
Java源文件(.java) → 编译器 → 字节码(.class) → JVM → 机器码
一次编写，到处运行
```

### 1.1.2 JVM架构

```
┌─────────────────────────────────────┐
│          类加载器（ClassLoader）      │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         运行时数据区（Runtime Data）   │
│  ┌────────────────────────────────┐ │
│  │ 程序计数器                      │ │
│  ├────────────────────────────────┤ │
│  │ 虚拟机栈                        │ │
│  ├────────────────────────────────┤ │
│  │ 本地方法栈                      │ │
│  ├────────────────────────────────┤ │
│  │ 堆（Heap）                      │ │
│  ├────────────────────────────────┤ │
│  │ 方法区                          │ │
│  └────────────────────────────────┘ │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         执行引擎（Execution Engine）  │
│    （解释器 + JIT编译器 + GC）        │
└─────────────────────────────────────┘
```

## 1.2 运行时数据区

### 1.2.1 程序计数器

```java
// 程序计数器（Program Counter Register）
// 每个线程独有，用于记录当前线程执行的字节码行号

// 作用：
// - 字节码解释器通过改变PC的值来选取下一条要执行的字节码指令
// - 分支、循环、跳转、异常处理都依赖程序计数器
// - 多线程环境下，每个线程需要独立的程序计数器

// 特点：
// - 线程私有
// - 唯一一个不会发生OutOfMemoryError的区域
// - 如果执行Native方法，计数器值为undefined
```

### 1.2.2 虚拟机栈

```java
// 虚拟机栈（VM Stack）
// 每个线程独有，存储方法调用的栈帧

// 栈帧（Stack Frame）结构：
// ┌─────────────────┐
// │  局部变量表      │ - 方法参数和局部变量
// ├─────────────────┤
// │  操作数栈        │ - 中间计算结果
// ├─────────────────┤
// │  动态链接        │ - 方法访问
// ├─────────────────┤
// │  方法返回地址     │ - 返回位置
// └─────────────────┘

// 栈深度过深
// StackOverflowError：线程请求的栈深度超过最大深度
// OutOfMemoryError：无法申请足够内存创建栈

// 常用参数
// -Xss1m      设置栈大小（1MB）
// -Xss256k    设置栈大小（256KB）
```

### 1.2.3 本地方法栈

```java
// 本地方法栈（Native Method Stack）
// 与虚拟机栈类似，但服务于Native方法

// 区别：
// - 虚拟机栈执行Java方法
// - 本地方法栈执行Native方法

// 有些JVM（如Sun HotSpot）将两者合二为一
```

### 1.2.4 堆（Heap）

```java
// 堆（Heap）
// 所有线程共享，存储对象实例和数组

// 分代结构（JDK 8+）：
// ┌───────────────────────────────────┐
// │              Old区                │
// │         (Tenured Space)           │
// ├───────────────────────────────────┤
// │  Survivor S1 │  From │    To      │
// │     (From)   │  S0   │    S1      │
// ├───────────────────────────────────┤
// │              Eden区               │
// └───────────────────────────────────┘

// 特点：
// - 线程共享
// - GC主要管理区域
// - 可分为新生代和老年代
// - 大小可以动态扩展（-Xms, -Xmx）

// 常用参数
// -Xms256m      初始堆大小
// -Xmx1024m     最大堆大小
// -Xmn512m      新生代大小
// -XX:NewRatio=2  老年代/新生代比例
```

### 1.2.5 方法区（MetaSpace）

```java
// 方法区（Method Area）
// 所有线程共享，存储类信息、常量、静态变量

// JDK 8之前：PermGen（永久代）
// JDK 8+：MetaSpace（元空间）

// 方法区存储内容：
// - 类信息（类的版本、字段、方法、接口等）
// - 运行时常量池
// - 静态变量
// - JIT编译产物

// 常用参数
// JDK 8之前：
// -XX:PermSize=128m
// -XX:MaxPermSize=256m

// JDK 8+：
// -XX:MetaspaceSize=256m
// -XX:MaxMetaspaceSize=512m
```

### 1.2.6 运行时常量池

```java
// 运行时常量池（Runtime Constant Pool）
// 方法区的一部分，存储编译期生成的常量

// 包含：
// - 字面量（字符串字面量、final常量）
// - 符号引用（类名、方法名、字段名）

// 示例
String s1 = "hello";
String s2 = "hello";
// s1和s2指向常量池中的同一个"hello"
```

## 1.3 对象的创建

### 1.3.1 对象创建过程

```java
// 对象创建过程：

// 1. 检查类加载
//    遇到new指令时，先检查常量池中是否有类的符号引用
//    检查这个类是否已被加载、解析、初始化

// 2. 分配内存
//    在堆中分配内存（对象头 + 实例数据）

// 内存分配方式：
// - 指针碰撞（Bump the Pointer）：堆规整，用过的放一边，没用的放另一边
// - 空闲列表（Free List）：堆不规整，维护可用内存块列表

// 线程安全问题：
// - CAS + 重试：保证原子性
// - TLAB（Thread Local Allocation Buffer）：每个线程预分配一块内存

// 3. 初始化零值
//    内存分配后，将内存初始化为零值
//    （不包括对象头）

// 4. 设置对象头
//    设置对象头信息（Mark Word、类型指针）

// 5. 执行<init>
//    执行构造函数，初始化真正值
```

### 1.3.2 对象的内存布局

```java
// 对象内存布局：
// ┌─────────────────────────────┐
// │        对象头（Header）      │
// │  ┌───────────────────────┐  │
// │  │   Mark Word (64bit)   │  │
// │  │  哈希码、GC分代年龄     │  │
// │  │  锁状态标志、偏向锁ID  │  │
// │  └───────────────────────┘  │
// │  ┌───────────────────────┐  │
// │  │   类型指针（Klass）    │  │
// │  │  指向类元数据的指针    │  │
// │  └───────────────────────┘  │
// │  ┌───────────────────────┐  │
// │  │   数组长度（如果是数组）│  │
// │  └───────────────────────┘  │
// ├─────────────────────────────┤
// │     实例数据（Instance Data）│
// │  父类字段 + 子类字段        │
// ├─────────────────────────────┤
// │   对齐填充（Padding）       │
// │  8字节对齐                  │
// └─────────────────────────────┘

// Mark Word 结构（64位JVM）：
// 对象未锁定：25位hashcode + 4位分代年龄 + 2位锁标志 + 1位偏向锁位
// 轻量级锁定：30位指向栈中锁记录的指针
// 重量级锁定：30位指向互斥量（monitor）的指针
// GC标记：空
// 可偏向：54位偏向线程ID + 2位偏向锁epoch + 4位分代年龄 + 2位锁标志
```

### 1.3.3 对象的访问定位

```java
// 对象访问方式：

// 1. 句柄访问（Handle）
// ┌─────────┐      ┌─────────┐      ┌─────────┐
// │  Java栈  │ ──→ │  句柄池  │ ──→ │  对象实例 │
// │ reference│      │  句柄    │      │  堆中   │
// └─────────┘      └─────────┘      └─────────┘
//                        ↓
//                   ┌─────────┐
//                   │类型数据  │
//                   │ 方法区   │
//                   └─────────┘
// 优点：对象移动时只需改变句柄中的指针

// 2. 直接指针访问（HSPOt）
// ┌─────────┐      ┌───────────────┐
// │  Java栈  │ ──→ │  对象实例      │
// │ reference│      │ ┌───────────┐ │
// └─────────┘      │ │ 对象头    │ │
//                   │ ├───────────┤ │
//                   │ │ 实例数据  │ │
//                   │ ├───────────┤ │
//                   │ │ 类型指针 ──┼─┼──→ 类型数据
//                   │ └───────────┘ │
//                   └───────────────┘
// 优点：访问速度快，HotSpot使用这种方式
```

## 1.4 垃圾回收概述

### 1.4.1 什么是垃圾

```java
// 垃圾：不再被引用的对象

// 引用计数法（不常用）
// 每个对象有个引用计数器，为0则回收
// 问题：循环引用无法回收

// 可达性分析（根搜索）
// 从GC Roots出发，向下搜索
// 不能到达的对象就是垃圾

// GC Roots包括：
// - 虚拟机栈（栈帧中的本地变量表）中引用的对象
// - 方法区中类静态属性引用的对象
// - 方法区中常量引用的对象
// - 本地方法栈中JNI引用的对象
// - JVM内部引用（Class对象、异常对象等）
// - 所有被同步锁（synchronized）持有的对象
```

### 1.4.2 四种引用

```java
// 强引用（Strong Reference）
Object obj = new Object();
// 强引用存在，对象永远不会被回收

// 软引用（Soft Reference）
SoftReference<Object> soft = new SoftReference<>(new Object());
// 内存不足时回收，第二次GC回收

// 弱引用（Weak Reference）
WeakReference<Object> weak = new WeakReference<>(new Object());
// 下一次GC时回收

// 虚引用（Phantom Reference）
// 形同虚设，无法通过虚引用访问对象
// 唯一作用：跟踪对象被GC回收的活动
// 必须配合ReferenceQueue使用

// 示例：缓存实现
Map<String, SoftReference<Cache>>> cache = new HashMap<>();
```

## 1.5 常用JVM参数

### 1.5.1 内存参数

```bash
# 堆内存
-Xms256m          # 初始堆大小
-Xmx1024m         # 最大堆大小
-Xmn128m          # 新生代大小
-XX:NewSize=128m  # 新生代初始大小
-XX:MaxNewSize=256m # 新生代最大大小
-XX:NewRatio=2    # 老年代/新生代比例（2表示老年代是新生代的2倍）
-XX:SurvivorRatio=8 # Eden/Survivor比例（8表示Eden是Survivor的8倍）

# 元空间（JDK 8+）
-XX:MetaspaceSize=256m  # 元空间初始大小
-XX:MaxMetaspaceSize=512m # 元空间最大大小

# 栈
-Xss1m            # 线程栈大小

# 直接内存
-XX:MaxDirectMemorySize=512m # 直接内存大小
```

### 1.5.2 GC参数

```bash
# 收集器选择
-XX:+UseSerialGC       # Serial收集器
-XX:+UseParallelGC     # Parallel收集器
-XX:+UseConcMarkSweepGC # CMS收集器
-XX:+UseG1GC          # G1收集器
-XX:+UseZGC           # ZGC（JDK 11+）

# Parallel GC参数
-XX:ParallelGCThreads=4  # Parallel GC线程数
-XX:MaxGCPauseMillis=100  # 最大GC停顿时间目标

# G1参数
-XX:MaxGCPauseMillis=200 # 最大GC停顿时间目标
-XX:G1HeapRegionSize=4m  # G1 region大小（1MB-32MB）
```

### 1.5.3 其他参数

```bash
# 日志
-Xlog:gc*:file=gc.log # 打印GC日志到文件

# 常见组合
# 生产环境推荐
-server
-Xms4g -Xmx4g
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
-Xlog:gc*:file=/var/log/myapp-gc.log

# 低延迟应用
-server
-Xms2g -Xmx2g
-XX:+UseZGC
-XX:+AlwaysPreTouch
```

## 1.7 参考答案

```java
// 1. JVM内存区域
// 线程私有：程序计数器、虚拟机栈、本地方法栈
// 线程共享：堆、方法区

// 2. 对象创建过程
// 检查类加载 → 分配内存（指针碰撞/空闲列表） → 初始化零值
// → 设置对象头 → 执行<init>初始化

// 3. GC Roots
// - 虚拟机栈中引用的对象
// - 方法区静态属性引用的对象
// - 方法区常量引用的对象
// - 本地方法栈JNI引用的对象
// - JVM内部引用
// - 同步锁持有的对象

// 4. 四种引用
// 强引用：永远不会回收
// 软引用：内存不足时回收
// 弱引用：下次GC时回收
// 虚引用：无法访问对象，用于跟踪GC活动

// 5. Metaspace vs PermGen
// PermGen: 有大小限制（-XX:MaxPermSize），固定连续内存
// Metaspace: 可用系统内存（-XX:MaxMetaspaceSize），动态扩展
// Metaspace避免 PermGen OOM 问题
```
